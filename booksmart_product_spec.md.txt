## 1. Executive Summary

### 1.1 Product Vision
BookSmart is an intelligent Chrome extension that revolutionizes bookmark management through AI-powered content understanding and semantic search. The system automatically captures, summarizes, and indexes bookmarked content, enabling users to find information based on meaning rather than just keywords.

### 1.2 Key Differentiators
- Zero friction - works with existing Chrome bookmark behavior
- AI-powered summaries using Google Gemini Flash
- Hybrid search combining semantic and full-text capabilities
- Extensible plugin architecture for future data sources
- Cost-effective at scale (<$0.10 per user per month)

### 1.3 Target Users
- Knowledge workers who bookmark frequently
- Researchers and students
- Content creators and writers
- Anyone who struggles to find previously saved content

### 1.4 Technical Architecture (Decided)
- **Vector Database**: Qdrant Cloud for semantic search
- **Backend Database**: Supabase (PostgreSQL + Auth)
- **API Hosting**: Vercel Functions
- **AI Processing**: Google Gemini Flash
- **MVP Cost**: $0/month (all services have free tiers)

## 2. Product Overview

### 2.1 Problem Statement
Current bookmark systems only save URLs and titles, making it impossible to find content when users can't remember exact titles or websites. Users accumulate hundreds of bookmarks that become effectively unsearchable.

### 2.2 Solution
BookSmart automatically:
- Captures full page content when bookmarking
- Generates AI summaries for quick understanding
- Enables search by concept, not just keywords
- Provides instant access through smart caching

### 2.3 Scope
**In Scope (MVP):**
- Chrome extension for bookmark capture
- AI summarization with Gemini Flash
- Hybrid search (semantic + full-text)
- Web-based manager interface
- Import tool for existing bookmarks

**Out of Scope (Initial Release):**
- ❌ Screenshots/thumbnails
- ❌ Smart collections
- ❌ Encryption layer
- ❌ Google Drive integration (future plugin)
- ❌ Cross-browser support (future)

## 3. Core Features

### 3.1 Smart Bookmark Capture

**Functional Requirements:**
- Detect Chrome bookmark events automatically
- Extract complete page content (HTML, PDF, Word docs)
- Intelligently detect and process image-centric pages
- Send to backend for processing
- Show non-intrusive success notification

**Supported Content Types:**
- Web pages (HTML)
- PDF documents
- Word documents (when displayed in browser)
- Image-centric pages (analyzed with Gemini Flash vision)

**Technical Implementation:**
- Chrome Extension Manifest V3
- Content script for extraction
- Smart image detection logic
- Background service worker for API communication

#### 3.1.1 Image Content Detection

**When to Process as Image Bookmark:**
- Direct image URLs (ends with .jpg, .png, .gif, .webp, .svg)
- Image hosting service pages (Imgur, Flickr, Instagram posts)
- Pages with <500 words of text AND single dominant image (>50% viewport)
- Pages where Open Graph type is "image" or "photo"
- Single-image viewer pages with minimal UI

**When NOT to Process Images:**
- Regular articles/blogs with embedded images
- E-commerce product pages (multiple product shots)
- Gallery pages with multiple images
- Any page with >500 words of text content
- News articles with supporting images

**Image Processing Limits:**
- Maximum 1 image per bookmark
- Image must be >200px in both dimensions
- Skip images larger than 10MB
- Process only the primary/largest image if criteria met

**Detection Priority:**
1. Check if URL is direct image link → Process as image
2. Check if known image hosting site → Extract main image
3. Analyze page content ratio → Process if image-dominant
4. Otherwise → Process as regular text bookmark

### 3.2 AI Processing Pipeline

**Components:**
1. **Summary Generation**
   - Model: Gemini Flash (not Pro)
   - Length: 150-300 words
   - Text content: ~$0.00003 per bookmark
   - Image content: ~$0.00002 per image

2. **Image Processing (when applicable)**
   - Only for image-centric pages (see Section 3.1.1)
   - Generate text description using Gemini Flash vision
   - Extract any text visible in image (OCR)
   - Create searchable tags from visual content
   - Examples: infographics, charts, diagrams, screenshots

3. **Tag Extraction**
   - Generate 5-10 relevant tags
   - Combine AI-suggested and user-defined tags
   - Include visual tags for image content

4. **Vector Embedding**
   - Dimension: 768 (not 1536 for cost savings)
   - Model: TBD (text-embedding-ada-002 or alternative)
   - Embed both text summaries and image descriptions

**Content Type Handling:**
| Content Type | Processing Method |
|--------------|-------------------|
| Regular webpage | Extract text, generate summary |
| Image-centric page | Analyze image with Gemini vision |
| PDF document | Extract text, process normally |
| Mixed content page | Process text only (ignore images) |
| Direct image URL | Pure image analysis |

**[DECISION NEEDED: Which embedding model to use?]**

### 3.3 Search System

**Search Modes:**
1. **Semantic Search** (60% weight)
   - Find conceptually similar content
   - Vector similarity using cosine distance

2. **Full-Text Search** (40% weight)
   - Exact phrase matching
   - Critical for technical content (error codes, function names)

**Why Hybrid Search is Essential:**
| Query Type | Example | Needs |
|------------|---------|--------|
| Conceptual | "Italian recipes" | Semantic |
| Technical | "useState React hook" | Full-text |
| Error codes | "Error 404" | Full-text |
| Visual content | "blue chart about revenue" | Semantic |
| General | "machine learning" | Both |

**Search Capabilities:**
- Search text content and summaries
- Search image descriptions and extracted text
- Find bookmarks by visual elements described by AI

## 4. Technical Architecture

### 4.1 System Architecture (Final)

```
┌─────────────────────────────────────┐
│     Chrome Extension (Frontend)      │
├─────────────────────────────────────┤
│  - Content Extraction                │
│  - Local Cache (IndexedDB)           │
│  - UI Components (React)             │
└────────────────┬────────────────────┘
                 │ HTTPS
┌────────────────▼────────────────────┐
│      Backend API Server              │
│      (Node.js on Vercel)             │
├─────────────────────────────────────┤
│  - Authentication Middleware         │
│  - Processing Pipeline               │
│  - Search Orchestration              │
│  - API Rate Limiting                 │
└────────────────┬────────────────────┘
                 │
    ┌────────────┴────────────┐
    │                          │
┌───▼────────┐      ┌──────────▼──────┐
│  Qdrant    │      │    Supabase     │
│   Cloud    │      ├─────────────────┤
├────────────┤      │ - PostgreSQL DB │
│  Vector    │      │ - Auth Service  │
│  Search    │      │ - File Storage  │
└────────────┘      └─────────────────┘

External Services:
- Google Gemini Flash API (AI processing)
- Embedding API (vector generation)
```

### 4.2 Technology Stack (Final)

**Frontend:**
- Chrome Extension Manifest V3
- React 18 (manager page)
- Tailwind CSS
- IndexedDB for local caching

**Backend:**
- Node.js with Express/Fastify
- Hosted on Vercel (free tier)
- JWT authentication via Supabase

**Databases:**
- **Qdrant Cloud**: Vector storage and semantic search
- **Supabase**: PostgreSQL, authentication, user management

**AI Services:**
- Google Gemini Flash (summaries and image analysis)
- Text embedding service (TBD: OpenAI or alternative)

### 4.3 API Architecture

**Core Endpoints:**
```
POST   /api/auth/register     → Supabase Auth
POST   /api/auth/login        → Supabase Auth
POST   /api/bookmarks         → Process & store in Qdrant + Supabase
GET    /api/bookmarks         → Retrieve from Qdrant
DELETE /api/bookmarks/:id     → Remove from both databases
GET    /api/search            → Query Qdrant + log in Supabase
POST   /api/bookmarks/import  → Batch import existing bookmarks
```

## 6. Data Storage Strategy

### 6.1 Database Architecture (FINAL DECISION)

**Selected Architecture: Qdrant Cloud + Supabase**

This architecture combines the best of both worlds: Qdrant's superior vector search with Supabase's excellent PostgreSQL and authentication services.

| Component | Service | Purpose | Free Tier Capacity |
|-----------|---------|---------|-------------------|
| **Vector Database** | Qdrant Cloud | Semantic search, bookmark storage | 500K vectors, 1GB RAM |
| **Relational Database** | Supabase PostgreSQL | User data, auth, preferences | 500MB storage |
| **Authentication** | Supabase Auth | User management, sessions | Unlimited users |
| **File Storage** | Supabase Storage | Future use (exports, etc.) | 2GB storage |

**MVP Cost: $0/month** (both services have generous free tiers)

### 6.2 Data Storage Distribution

**Qdrant Cloud Stores:**
- Vector embeddings (768 dimensions)
- Full bookmark content and summaries
- Image descriptions and extracted text
- Tags and metadata
- All searchable bookmark data in payload
- User ID for filtering

**Data Structure in Qdrant:**
```
{
  "id": "bookmark-uuid",
  "vector": [768-dimensional array],
  "payload": {
    "user_id": "from-supabase-auth",
    "url": "https://...",
    "title": "Page Title",
    "summary": "AI-generated summary",
    "content": "First 1000 words",
    "tags": ["tag1", "tag2"],
    "image_description": "If image bookmark",
    "created_at": "2024-01-01"
  }
}
```

**Supabase PostgreSQL Stores:**
- User accounts (managed by Supabase Auth)
- Authentication tokens and sessions
- Bookmark references (id, user_id, url, created_at)
- Search history and analytics
- User preferences and settings
- Import job tracking
- System audit logs

**Database Tables in Supabase:**
```
bookmarks: Lightweight reference table
search_history: Track user searches
user_preferences: Settings and configurations
import_jobs: Track bulk import progress
```

**IndexedDB (Local Cache):**
- Recent 100 bookmarks for offline access
- Search result cache (1-hour TTL)
- User preferences mirror
- Temporary processing queue

## 8. Cost Analysis

### 8.1 MVP Costs (Qdrant + Supabase Architecture)

**Database Costs for MVP: $0/month**

| Service | Free Tier Includes | Capacity | When You'll Pay |
|---------|-------------------|----------|-----------------|
| **Qdrant Cloud** | 1GB cluster, unlimited API calls | 500K bookmarks | After 500K vectors (~$25/mo) |
| **Supabase** | 500MB PostgreSQL, Auth, 2GB storage | 1000+ users | After 500MB (~$25/mo) |
| **Vercel** | API hosting, 100GB bandwidth | 10K+ API calls/day | After 100GB (~$20/mo) |

**Only Cost for MVP:**
- Gemini Flash API: ~$0.00003 per bookmark
- Total for 10,000 bookmarks: ~$0.30

### 8.2 Scaling Costs

**Storage Costs by User Scale:**
| Users | Bookmarks | Qdrant | Supabase | Gemini API | Total/Month |
|-------|-----------|---------|----------|------------|-------------|
| 100 | 10,000 | $0 (free) | $0 (free) | ~$0.30 | **$0** |
| 1,000 | 200,000 | $0 (free) | $0 (free) | ~$6 | **$0** |
| 2,500 | 500,000 | $0 (free) | $0 (free) | ~$15 | **$0** |
| 5,000 | 1,000,000 | $25 | $25 | ~$30 | **$80** |
| 10,000 | 2,000,000 | $95 | $25 | ~$60 | **$180** |

### 8.4 Pricing Strategy Options

**Recommended Pricing (Based on Cost Analysis):**

**Freemium Model:**
- **Free Tier**: 100 bookmarks (costs us $0)
- **Pro**: $3/month for unlimited (profitable from day 1)
- **Team**: $10/user/month (future)

**Why This Works:**
- Free tier costs nothing to provide
- Paid tier has 97% profit margin at scale
- Can support 2,500 free users before any database costs
- Break-even with just 27 paying users (at 1000 total users)

**Alternative - Paid Only:**
- $2/month for all users
- Simpler model
- Break-even at ~40 users

### 8.5 Architecture Decision Rationale

**Why Qdrant + Supabase (Final Decision):**
1. **Best Performance**: Qdrant excels at vector search
2. **Complete Solution**: Supabase provides auth + PostgreSQL + storage
3. **Free MVP**: Both services free up to thousands of users
4. **Clear Separation**: Each service does what it's best at
5. **Proven Scale**: Both can handle millions of users
